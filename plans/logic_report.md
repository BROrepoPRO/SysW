## Mod_Utils.bas

**Назначение:** Вспомогательные (утилитарные) функции для работы с Excel-книгой.

### Типы данных

#### `OrderHeader` (Public Type)
Пользовательский тип для хранения данных заказа-наряда. Поля соответствуют столбцам листа `spisok`:

| Поле | Тип | Столбец spisok | Описание |
|------|-----|----------------|----------|
| `OrderNumber` | String | A | № п/п |
| `ModelName` | String | B | Модель |
| `GRZ` | String | C | ГРЗ/госномер |
| `VIN` | String | D | VIN |
| `GarageNumber` | String | E | Гаражный № |
| `YearMade` | Integer | F | Год выпуска |
| `MileageValue` | Long | G | Пробег |
| `DateValue` | Date | H | Дата |

### Функции

#### `GetSheetByName(wb, SheetName) As Worksheet`
- **Назначение:** Получение листа по имени без генерации ошибки, если лист не найден.
- **Вход:** `wb` — Workbook, `SheetName` — String (имя листа).
- **Выход:** Объект Worksheet или `Nothing`, если лист не существует.
- **Логика:** Использует `On Error Resume Next` для подавления ошибки при обращении к несуществующему листу.

#### `GetWorkbookPath() As String`
- **Назначение:** Получение пути к текущей книге.
- **Вход:** Нет.
- **Выход:** String — путь к папке, где находится `ThisWorkbook`.
- **Логика:** Возвращает `ThisWorkbook.Path`.

#### `FileExists(FilePath) As Boolean`
- **Назначение:** Проверка существования файла на диске.
- **Вход:** `FilePath` — String (полный путь к файлу).
- **Выход:** Boolean — `True`, если файл существует.
- **Логика:** Использует `Dir(FilePath)` и проверяет длину результата > 0.

#### `GetCurrentUser() As String`
- **Назначение:** Получение имени текущего пользователя Windows.
- **Вход:** Нет.
- **Выход:** String — имя пользователя.
- **Логика:** Возвращает `Environ("USERNAME")`.

#### `FormatDateSQL(d) As String`
- **Назначение:** Форматирование даты в формат SQLite (ГГГГ-ММ-ДД).
- **Вход:** `d` — Date.
- **Выход:** String — дата в формате `"yyyy-mm-dd"`.
- **Логика:** Использует `Format(d, "yyyy-mm-dd")`.

### Процедуры

#### `WriteLog(Message)`
- **Назначение:** Запись сообщения в лог-файл.
- **Вход:** `Message` — String (текст сообщения).
- **Выход:** Нет (запись в файл).
- **Пишет:** Файл `log.txt` в папке книги.
- **Логика:** Открывает `log.txt` для добавления, пишет строку вида `[дата время] - [сообщение]`.

## Mod_OrderHeader.bas

**Назначение:** Автоматическое заполнение полей заказа-наряда на листе `main` из данных листов `spisok` и `model`.

### Процедуры

#### `FillHeaderFromOrder(OrderNum, wsMain, wsSpisok, wsModel)`
- **Назначение:** Заполняет ячейки B3:B15 на листе `main` данными из найденного заказа.
- **Вход:**
  - `OrderNum` — String (номер заказа для поиска, обычно значение из B2).
  - `wsMain` — Worksheet (лист `main`).
  - `wsSpisok` — Worksheet (лист `spisok`).
  - `wsModel` — Worksheet (лист `model`).
- **Выход:** Нет (запись в ячейки).
- **Читает:**
  - Лист `spisok`, колонка A — поиск по `OrderNum` (xlWhole).
  - Лист `spisok`, колонки A–J (смещения 0..9 от найденной ячейки).
  - Лист `model`, колонка A — поиск по коду модели (из колонки I листа `spisok`).
  - Лист `model`, колонки A–E (смещения 0..4 от найденной ячейки).
- **Пишет (лист `main`):**

  | Ячейка main | Источник | Столбец spisok | Описание |
  |-------------|----------|----------------|----------|
  | B3 | Формула: `"00" & B2 & "-20"` | — | Номер заказ-наряда |
  | B4 | `FoundCell.Offset(0, 1)` | B | Модель |
  | B5 | `FoundCell.Offset(0, 2)` | C | ГРЗ |
  | B6 | `FoundCell.Offset(0, 3)` | D | VIN |
  | B7 | `FoundCell.Offset(0, 4)` | E | Гараж. № |
  | B8 | `FoundCell.Offset(0, 5)` | F | Год вып. |
  | B9 | `FoundCell.Offset(0, 6)` | G | Пробег |
  | B10 | `FoundCell.Offset(0, 7)` | H | Дата |
  | B11 | `ModelFound.Offset(0, 2)` | C (model) | Название модели |
  | B12 | `ModelFound.Offset(0, 1)` | B (model) | Кузов |
  | B13 | `ModelFound.Offset(0, 3)` | D (model) | Год выпуска (из model) |
  | B14 | `ModelFound.Offset(0, 4)` | E (model) | VIN (из model) |
  | B15 | `FoundCell.Offset(0, 9)` | J (spisok) | Примечание |

- **Логика:**
  1. Проверяет, что `wsSpisok` и `wsModel` не `Nothing`.
  2. Ищет `OrderNum` в колонке A листа `spisok`.
  3. Если найден — заполняет B3:B15 по маппингу выше.
  4. Дополнительно ищет код модели (колонка I листа `spisok`) в колонке A листа `model` и заполняет B11:B14.
  5. Если не найден — очищает B3:B15.

#### `FindOrder(OrderNum, Header) As Boolean`
- **Назначение:** Поиск заказа по № п/п и заполнение структуры `OrderHeader`.
- **Вход:**
  - `OrderNum` — String (номер заказа для поиска).
  - `Header` — `OrderHeader` (передаётся по ссылке, заполняется).
- **Выход:** Boolean — `True`, если заказ найден.
- **Читает:** Лист `spisok`, колонка A (поиск), колонки A–H (заполнение полей).
- **Логика:**
  1. Получает лист `spisok` через `GetSheetByName`.
  2. Ищет `OrderNum` в колонке A.
  3. Если найден — заполняет поля `Header` из колонок A–H.
  4. Возвращает `True/False`.

#### `ImportSheet(GRZ)`
- **Назначение:** Импорт данных с листа по госномеру.
- **Вход:** `GRZ` — String (госномер).
- **Выход:** Нет (запись в лист `main`).
- **Логика:** Находит лист через `SearchSheetByGRZ` и вызывает `ImportDataToMain`.

#### `ImportIncomingDocNumber(GRZ)`
- **Назначение:** Импорт входящих номеров документов.
- **Вход:** `GRZ` — String (госномер).
- **Выход:** Нет (запись в лист `main`).
- **Читает:** Лист источника, колонка A (начиная со строки 2).
- **Пишет:** Лист `main`, колонка W (последняя пустая строка).
- **Логика:**
  1. Находит лист по госномеру.
  2. Проходит по колонке A (строки 2..LastRow).
  3. Копирует непустые значения в колонку W листа `main`.

#### `ImportDataToMain(wsSource)`
- **Назначение:** Импорт данных из листа-источника в лист `main` с маппингом столбцов.
- **Вход:** `wsSource` — Worksheet (лист-источник).
- **Выход:** Нет (запись в лист `main`).
- **Читает:** Лист-источник, колонки B, C, D, G, H (со строки 2 до LastRow по колонке B).
- **Пишет (лист `main`):**

  | Столбец источника | Столбец main | Описание |
  |-------------------|-------------|----------|
  | C | L | Данные |
  | D | M | Данные |
  | H | N | Данные |
  | B | X | Данные |
  | C | Y | Данные |
  | D | Z | Данные |
  | G | AA | Данные |

- **Логика:**
  1. Определяет последнюю строку в источнике по колонке B.
  2. Для каждой строки (2..LastRow), если колонка B не пуста:
     - Определяет следующую пустую строку в main по колонке L.
     - Копирует данные по маппингу.
  3. Показывает сообщение о количестве обработанных строк.

#### `ImportFromReport()`
- **Назначение:** Импорт данных из листа `report` в лист `main`.
- **Вход:** Нет (работает с `ThisWorkbook`).
- **Выход:** Нет (запись в лист `main`).
- **Читает:** Лист `report`, колонки A, B, C, D (со строки 2 до LastRow по колонке A).
- **Пишет (лист `main`):**

  | Столбец report | Столбец main | Описание |
  |----------------|-------------|----------|
  | A | A | Данные (например, ГРЗ) |
  | B | B | Данные (например, марка) |
  | C | C | Данные (например, модель) |

  > **Важно:** Колонка D из `report` **не импортируется** (игнорируется).

- **Логика:**
  1. Проверяет существование листа `report` через `GetSheetByName`.
  2. Определяет последнюю строку в `report` по колонке A.
  3. Для каждой строки (2..LastRow), если колонка A не пуста:
     - Определяет следующую пустую строку в `main` по колонке A.
     - Копирует A→A, B→B, C→C.
  4. Колонка D (год/доп. данные) из `report` **не переносится**.

## Mod_FullTestRunner.bas

**Назначение:** Полный прогон тестов для модулей SysW (TC-01..TC-20).

### Публичные процедуры

#### `RunAllTests()`
- **Назначение:** Запуск всех тестов.
- **Логика:**
  1. Инициализирует счётчики (Total=0, Passed=0, Failed=0, Skipped=0).
  2. Последовательно запускает: `RunUtilsTests`, `RunOrderHeaderTests`, `RunImportTests`, `RunButtonTests`.
  3. Выводит итоговый отчёт через `PrintFinalReport`.

### Приватные процедуры

#### `RunUtilsTests()`
- **Тесты:** TC-01..TC-04, TC-06, TC-07, TC-19, TC-20.
- **Проверяет:** `FileExists` (существующий/несуществующий файл), `FormatDateSQL` (нормальная/нулевая дата), `GetSheetByName` (существующий/несуществующий лист), `WriteLog`, `GetWorkbookPath`/`GetCurrentUser`.

#### `RunOrderHeaderTests()`
- **Тесты:** TC-08, TC-09, TC-11, TC-12.
- **Проверяет:** `FindOrder` (существующий/несуществующий заказ), `FillHeaderFromOrder` (с Nothing-параметрами, с ненайденным заказом — проверяет очистку B3:B15).

#### `RunImportTests()`
- **Тесты:** TC-05, TC-13, TC-14, TC-15, TC-17.
- **Проверяет:** `ExtractNumberFromGRZ` (одиночный/множественные тесты), `SearchSheetByGRZ` (существующий/несуществующий), `ImportFromReport` (создаёт временный лист `report`, импортирует, проверяет заполнение L2, восстанавливает состояние).

#### `RunButtonTests()`
- **Тесты:** TC-18.
- **Проверяет:** `Btn_main_Clear_Click` — только заглушка (пропущен из-за MsgBox).

### Вспомогательные процедуры

#### `SaveSheetRange(ws, RangeAddr) As Variant`
- Сохраняет значения диапазона для последующего восстановления.

#### `RestoreSheetRange(ws, RangeAddr, data)`
- Восстанавливает сохранённые значения диапазона.

#### `AddResult(testId, testName, passed, failReason, skipped, skipReason)`
- Добавляет результат теста, выводит в Immediate Window.

#### `PrintFinalReport()`
- Выводит итоговую статистику в Immediate Window и через MsgBox.

## Sheet1_main.cls

**Назначение:** Code-behind листа `main` — обработка событий листа.

### События

#### `Worksheet_Change(ByVal Target As Range)`
- **Назначение:** Автоматический запуск `FillHeaderFromOrder` при изменении ячейки B2.
- **Вход:** `Target` — Range (изменённый диапазон).
- **Выход:** Нет (запись в ячейки B3:B15).
- **Логика:**
  1. **Защита от рекурсии:** Использует `Static IsProcessing As Boolean`.
  2. **Проверки:**
     - `Application.EnableEvents` должен быть `True`.
     - `Target.CountLarge` должен быть 1 (только одна ячейка).
     - `Target` должен пересекаться с `Me.Range("B2")`.
     - `Target` **не должен** пересекаться с `Me.Range("B3:B15")` (чтобы не было цикла при программном заполнении).
  3. **Действие:**
     - Отключает события (`Application.EnableEvents = False`).
     - Получает листы `spisok` и `model` через `GetSheetByName`.
     - Вызывает `Mod_OrderHeader.FillHeaderFromOrder Trim(Target.Value), Me, wsSpisok, wsModel`.
     - Восстанавливает события в блоке `CleanUp`.
  4. **Ошибки:** Любая ошибка приводит к переходу в `CleanUp` (восстановление событий).

## Сводка по потокам данных

### Импорт из отчёта (`ImportFromReport`)
```
Лист "report" (A, B, C, D)  ──→  Лист "main" (A, B, C)
  A → A (ГРЗ/номер)
  B → B (марка/модель)
  C → C (модель/описание)
  D → ✗ (не импортируется!)
```

### Заполнение шапки заказа (`FillHeaderFromOrder`)
```
Лист "spisok" (A..J)  ──→  Лист "main" (B3:B15)
  A (№ п/п)       → B3 (через формулу "00"&B2&"-20")
  B (Модель)      → B4
  C (ГРЗ)         → B5
  D (VIN)         → B6
  E (гараж.№)     → B7
  F (год вып.)    → B8
  G (пробег)      → B9
  H (дата)        → B10
  I (код модели)  → поиск в model
  J (примечание)  → B15

Лист "model" (A..E)  ──→  Лист "main" (B11:B14)
  A (код модели)  → поиск по I из spisok
  B (кузов)       → B12
  C (название)    → B11
  D (год выпуска) → B13
  E (VIN)         → B14
```

### Импорт данных с листа-источника (`ImportDataToMain`)
```
Лист-источник (B, C, D, G, H)  ──→  Лист "main" (L, M, N, X, Y, Z, AA)
  B → X
  C → L, Y
  D → M, Z
  G → AA
  H → N
```

### Очистка (`Btn_main_Clear_Click`)
```
Лист "main": A2:XFD<LastRow> → ClearContents
(строка 1 — заголовки — не очищается)
```

## Ключевые замечания

1. **`ImportFromReport`** импортирует только колонки A, B, C из листа `report` в A, B, C листа `main`. Колонка D (год/доп. данные) теряется.
2. **`FillHeaderFromOrder`** использует колонку I листа `spisok` как код модели для поиска в листе `model` (колонка A). Маппинг model: A=код, B=кузов, C=название, D=год, E=VIN.
3. **`Btn_main_Clear_Click`** очищает все данные со строки 2 до последней использованной строки по всем колонкам (A–XFD).
4. **`Btn_main_Import_Click`** вызывает только `ImportFromReport` (не `ImportSheet` или `ImportDataToMain`).
5. **`Worksheet_Change`** срабатывает только при изменении одной ячейки B2 (не B3:B15) и вызывает `FillHeaderFromOrder`.
6. **Тесты TC-17** проверяют `ImportFromReport` через колонку L (а не A), что **не соответствует** реальной логике `ImportFromReport` (которая пишет в A, B, C). Это потенциальная ошибка в тесте.
